## COSA È STATO FATTO

### 1. NUOVO TRANSCEIVER (position_tracker_v6_final.py)

**Strategia di acquisizione dati:**

```
Ricevi telemetria dal transponder (ogni ~2s)
    ↓
[STEP 1] Query AT?T (timeout 0.3s)
    ↓ SEMPRE eseguito
    Ottieni: prop_time_us
    Calcola: range_3d = prop_time_us * speed_of_sound
    Tempo: ~100-150ms
    ↓
[STEP 2] Query USBLANGLES (timeout 1.2s)
    ↓ Eseguito solo se tempo_totale < 1.5s
    Ottieni: bearing_deg, elevation_deg, integrity
    Tempo: ~300-500ms
    ↓
[DECISIONE MODE]
    Se USBLANGLES OK + integrity≥50:
        → MODE = ANGLES
        → Calcola ENU da angles + range_3d
        → Invia: mode, bearing, elevation, ENU, range_3d
    
    Altrimenti:
        → MODE = RANGE
        → Invia: mode, range_3d (bearing=0, elevation=0, ENU=0)
    ↓
Invia risposta al transponder
Tempo totale: 400-900ms (ben sotto i 2s!)
```

**Tempi garantiti:**
- Caso migliore: ~400ms (AT?T + USBLANGLES successo)
- Caso tipico: ~600-900ms
- Caso peggiore: ~900ms (AT?T OK, USBLANGLES timeout)
- SEMPRE sotto 1.5s → margine sicuro per ciclo 2s

**Logging CSV:**
- File: `Transciever/log_usbl_messages/tc_rx_tp_TIMESTAMP.csv` (RX)
- File: `Transciever/log_usbl_messages/tc_tx_tp_TIMESTAMP.csv` (TX)
- Enable/disable: variabile `ENABLE_LOGGING` (riga 42)

```
Dati disponibili:
- range_3d: distanza 3D da AT?T (SEMPRE)
- bearing: angolo orizzontale in gradi
- elevation: angolo verticale in gradi
- ENU: calcolato da angles + range_3d
- integrity: qualità dati USBLANGLES

Accuratezza: ±0.5-2m
Quando: USBLANGLES risponde con integrity≥50
Frequenza attesa: 70-90% dei messaggi (ambiente controllato)
```

### MODE 2 - RANGE (fallback)
```
Dati disponibili:
- range_3d: distanza 3D da AT?T (SEMPRE)
- bearing: 0
- elevation: 0
- ENU: (0, 0, 0)
- integrity: default (100)

Accuratezza: ±0.1m solo in distanza, no posizione
Quando: USBLANGLES timeout o integrity<50
Frequenza attesa: 10-30% dei messaggi
```

## FORMULA CALCOLO ENU DA ANGLES

```python
# Input:
bearing_deg = 45.0      # gradi (0-360)
elevation_deg = 10.0    # gradi (-90 a +90)
range_3d = 0.75         # metri

# Conversione:
bearing_rad = math.radians(bearing_deg)
elevation_rad = math.radians(elevation_deg)

# Calcolo ENU (coordinate sferiche → cartesiane):
east = range_3d * cos(elevation_rad) * sin(bearing_rad)
north = range_3d * cos(elevation_rad) * cos(bearing_rad)
up = range_3d * sin(elevation_rad)

# Output esempio:
east = 0.522 m
north = 0.522 m
up = 0.130 m
```
